# 645. Set Mismatch

## Intuition

The array should contain every number from `1` to `n` exactly once. Due to an error, one number appears twice and another number is missing.

We can use a set to keep track of numbers we've already seen:

- If a number is already present in the set, it is the duplicate.
- After identifying the duplicate, iterate from `1` to `n` and find the number that is not present in the set. That number is the missing value.

## Approach

1. Create an empty set `seen`.
2. Traverse the array:
   - If the current number is already in the set, store it as the duplicate.
   - Otherwise, add it to the set.
3. Traverse numbers from `1` to `n`:
   - The first number not present in the set is the missing number.
4. Return `[duplicate, missing]`.

## Complexity Analysis

- **Time Complexity:** `O(n)`
  - One pass to find the duplicate.
  - One pass to find the missing number.

- **Space Complexity:** `O(n)`
  - The set stores up to `n` elements.

## Solution

```python
class Solution(object):
    def findErrorNums(self, nums):
        """
        :type nums: List[int]
        :rtype: List[int]
        """
        seen = set()
        duplicate = -1

        for num in nums:
            if num in seen:
                duplicate = num
            else:
                seen.add(num)

        missing = -1

        for i in range(1, len(nums) + 1):
            if i not in seen:
                missing = i
                break

        return [duplicate, missing]
```
