# 636. Exclusive Time of Functions

## Intuition

A function's exclusive time is the time spent executing its own code, excluding the time spent in functions that it calls.

Since function calls can be nested, a stack is the ideal data structure:

- When a function starts, push it onto the stack.
- When another function starts, the currently running function is paused.
- When a function ends, calculate its execution time and remove it from the stack.
- Keep track of the previous timestamp to determine how much time the currently active function has been running.

## Approach

1. Initialize a result array `ans` of size `n` with all zeros.
2. Create an empty stack to track active functions.
3. Maintain a variable `prev_time` representing the start of the current execution interval.
4. Process each log entry:
   - Parse the function id, action (`start` or `end`), and timestamp.
5. If the action is `"start"`:
   - If another function is currently running, add the elapsed time to its exclusive time.
   - Push the new function onto the stack.
   - Update `prev_time`.
6. If the action is `"end"`:
   - Pop the current function from the stack.
   - Add its execution time (`timestamp - prev_time + 1`) to the result.
   - Update `prev_time` to `timestamp + 1`.
7. Return the result array.

## Complexity Analysis

- **Time Complexity:** `O(m)`
  - Each log is processed exactly once.

- **Space Complexity:** `O(n)`
  - The stack can contain up to `n` nested function calls.

Where:

- `n` = number of functions
- `m` = number of log entries

## Solution

```python
class Solution(object):
    def exclusiveTime(self, n, logs):
        """
        :type n: int
        :type logs: List[str]
        :rtype: List[int]
        """
        ans = [0] * n
        stack = []
        prev_time = 0

        for log in logs:
            func_id, action, timestamp = log.split(":")
            func_id = int(func_id)
            timestamp = int(timestamp)

            if action == "start":
                if stack:
                    ans[stack[-1]] += timestamp - prev_time

                stack.append(func_id)
                prev_time = timestamp

            else:
                ans[stack.pop()] += timestamp - prev_time + 1
                prev_time = timestamp + 1

        return ans
```

## Why `timestamp + 1`?

The problem defines end timestamps as **inclusive**.

Example:

```text
start = 2
end = 5
```

The function runs during:

```text
2, 3, 4, 5
```

which is:

```text
5 - 2 + 1 = 4 units
```

After processing an end event at time `5`, the next available execution time starts at:

```text
6
```

Therefore:

```python
prev_time = timestamp + 1
```

## Interview Explanation

> Since function calls can be nested, we use a stack to track the currently executing functions. Whenever a new function starts, we pause the current function and record the time it has executed so far. When a function ends, we calculate its inclusive execution time and remove it from the stack. By maintaining a `prev_time` pointer and updating the active function's execution time whenever a context switch occurs, we can compute exclusive times in a single pass through the logs. This results in `O(m)` time complexity and `O(n)` space complexity.
