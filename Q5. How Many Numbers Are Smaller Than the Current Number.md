# 1365. How Many Numbers Are Smaller Than the Current Number

## Intuition

For each number in the array, count how many other numbers are smaller than it.

A straightforward way to do this is to compare each element with every other element in the array. Whenever we find a smaller number, we increment a counter.

After checking all elements against the current number, store the count in the result array.

## Approach

1. Initialize an empty result array.
2. Traverse the array using index `i`.
3. For each element, traverse the array again using index `j`.
4. Skip the comparison when `i == j`.
5. If `nums[j]` is smaller than `nums[i]`, increment the counter.
6. Append the counter to the result array.
7. Reset the counter and repeat for the next element.
8. Return the result array.

## Complexity Analysis

- **Time Complexity:** `O(n²)`
  - For every element, we compare it with every other element.

- **Space Complexity:** `O(n)`
  - The output array stores one result for each input element.

## Solution

```python
class Solution(object):
    def smallerNumbersThanCurrent(self, nums):
        """
        :type nums: List[int]
        :rtype: List[int]
        """
        smaller_count = 0
        output_list = []

        for i in range(len(nums)):
            for j in range(len(nums)):
                if i == j:
                    continue
                elif nums[i] > nums[j]:
                    smaller_count += 1

            output_list.append(smaller_count)
            smaller_count = 0

        return output_list
```
## ~~~~ One More Way to solve ~~~~

## Intuition

Instead of comparing every number with every other number, we can sort the array.

In the sorted array, the index of the first occurrence of a number represents how many numbers are smaller than it.

For example:

```text
nums = [8,1,2,2,3]

sorted_nums = [1,2,2,3,8]
```

The first occurrence indices are:

```text
1 -> 0
2 -> 1
3 -> 3
8 -> 4
```

These indices directly represent the count of smaller numbers.

## Approach

1. Create a sorted copy of the array.
2. Store the first occurrence index of each number in a dictionary.
3. Traverse the original array.
4. For each number, use the dictionary to find how many numbers are smaller than it.
5. Return the result.

## Complexity Analysis

- **Time Complexity:** `O(n log n)`
  - Sorting takes `O(n log n)`.
  - Dictionary construction and lookup take `O(n)`.

- **Space Complexity:** `O(n)`
  - Dictionary and output array require additional space.

## Solution

```python
class Solution(object):
    def smallerNumbersThanCurrent(self, nums):
        """
        :type nums: List[int]
        :rtype: List[int]
        """
        sorted_nums = sorted(nums)

        rank = {}

        for i, num in enumerate(sorted_nums):
            if num not in rank:
                rank[num] = i

        return [rank[num] for num in nums]
```

## ~~~~ One More Way to solve ~~~~

## Intuition

The constraints tell us that:

```text
0 <= nums[i] <= 100
```

This means there are only **101 possible values**.

Instead of comparing every number with every other number or sorting the array, we can count how many times each number appears.

Once we know the frequency of every number, we can build a prefix sum array where each index stores the count of numbers less than or equal to that value.

For any number `x`, the number of elements smaller than it is simply the count of all elements up to `x - 1`.

## Approach

1. Create a counting array of size `101` to store the frequency of each number.
2. Traverse `nums` and update the frequency count.
3. Convert the frequency array into a prefix sum array.
   - After this step, `count[i]` represents the number of elements less than or equal to `i`.
4. Traverse `nums` again:
   - If the current number is `0`, there are no smaller numbers.
   - Otherwise, the answer is `count[num - 1]`.
5. Store the results and return the output array.

## Complexity Analysis

- **Time Complexity:** `O(n + 101)`
  - One pass to build the frequency array.
  - One pass to build the prefix sums.
  - One pass to generate the result.

Since `101` is a constant:

- **Time Complexity:** `O(n)`

- **Space Complexity:** `O(101)`
  - The counting array always has a fixed size of `101`.

Since `101` is constant:

- **Space Complexity:** `O(1)`

## Solution

```python
class Solution(object):
    def smallerNumbersThanCurrent(self, nums):
        """
        :type nums: List[int]
        :rtype: List[int]
        """
        count = [0] * 101

        # Store frequency of each number
        for num in nums:
            count[num] += 1

        # Convert frequency array into prefix sum array
        for i in range(1, 101):
            count[i] += count[i - 1]

        result = []

        # Find count of smaller numbers
        for num in nums:
            if num == 0:
                result.append(0)
            else:
                result.append(count[num - 1])

        return result
```
