# 739. Daily Temperatures

## Intuition

For each day, we need to find:

> How many days do we have to wait until a **warmer temperature**?

This is equivalent to finding the **Next Greater Element to the Right**.

For every temperature, we need the first temperature on its right that is greater than it.

This is a classic **Monotonic Stack** problem.

---

## Approach

We'll use a **Monotonic Decreasing Stack** that stores the **indices** of temperatures.

Why indices instead of values?

Because the answer requires:

```text
number of days to wait = next_warmer_day_index - current_day_index
```

### Algorithm

1. Create an answer array initialized with zeros.
2. Create an empty stack to store indices.
3. Traverse the temperatures from left to right.
4. While:
   - the stack is not empty, and
   - the current temperature is greater than the temperature at the top index of the stack,
   
   we have found the next warmer day for the top index.

5. Pop the index and calculate:

```text
current_index - popped_index
```

6. Store the result in the answer array.
7. Push the current index onto the stack.
8. Return the answer array.

---

## Why Does This Work?

The stack stores indices whose next warmer temperature has not yet been found.

Whenever:

```python
temperatures[i] > temperatures[stack[-1]]
```

the current day is the first warmer day for the index at the top of the stack.

Therefore:

```python
answer[index] = i - index
```

## Solution

```python
class Solution(object):
    def dailyTemperatures(self, temperatures):
        """
        :type temperatures: List[int]
        :rtype: List[int]
        """
        n = len(temperatures)
        answer = [0] * n
        stack = []

        for i in range(n):
            while stack and temperatures[i] > temperatures[stack[-1]]:
                index = stack.pop()
                answer[index] = i - index

            stack.append(i)

        return answer
```

---

## Complexity Analysis

### Time Complexity: `O(n)`

Each index is:

- pushed onto the stack once,
- popped from the stack once.

Therefore:

```text
Total operations = 2n
                 = O(n)
```

---

### Space Complexity: `O(n)`

In the worst case:

```text
temperatures = [80,79,78,77,76]
```

No index is ever popped, so the stack stores all indices.

```text
Space Complexity = O(n)
```
