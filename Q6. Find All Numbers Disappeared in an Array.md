# 448. Find All Numbers Disappeared in an Array

## Intuition

The array contains numbers in the range `1` to `n`, where `n` is the length of the array.

If we store all the numbers present in the array inside a set, we can quickly check whether a number exists or not.

After building the set, we iterate through all numbers from `1` to `n`. Any number that is not present in the set must be missing from the array.

## Approach

1. Create an empty set to store the unique numbers present in the array.
2. Traverse the input array and add each number to the set.
3. Create an empty result array to store the missing numbers.
4. Iterate through all numbers from `1` to `n`.
5. If a number is not present in the set, add it to the result array.
6. Return the result array.

## Complexity Analysis

- **Time Complexity:** `O(n)`
  - One pass to build the set.
  - One pass to find the missing numbers.
  - Set lookups are `O(1)` on average.

- **Space Complexity:** `O(n)`
  - The set may store up to `n` unique elements.
  - The output array may also contain up to `n` elements in the worst case.

## Solution

```python
class Solution(object):
    def findDisappearedNumbers(self, nums):
        """
        :type nums: List[int]
        :rtype: List[int]
        """
        nums_set = set()
        missing_nums = []

        for num in nums:
            if num in nums_set:
                continue
            else:
                nums_set.add(num)

        for i in range(1, len(nums) + 1):
            if i not in nums_set:
                missing_nums.append(i)

        return missing_nums
```

# Approach 2

## Intuition

The array contains numbers in the range `1` to `n`, where `n` is the length of the array.

Since every number corresponds to a valid index (`num - 1`), we can use the input array itself to keep track of which numbers have been seen.

For every number:

- Take its absolute value.
- Use `abs(num) - 1` as an index.
- Mark the element at that index as negative.

After processing all numbers:

- If an index contains a positive value, it means that index was never visited.
- Therefore, the corresponding number (`index + 1`) is missing from the array.

This allows us to find the missing numbers without using any extra data structures.

## Approach

1. Traverse the array.
2. For each number:
   - Compute the index `abs(num) - 1`.
   - Mark the value at that index as negative if it is not already negative.
3. Traverse the array again.
4. For every positive value found at index `i`:
   - Add `i + 1` to the result array.
5. Return the result array.

## Complexity Analysis

- **Time Complexity:** `O(n)`
  - One pass to mark visited numbers.
  - One pass to find missing numbers.

- **Space Complexity:** `O(1)`
  - No extra data structure is used.
  - The modifications are performed directly on the input array.
  - The output array is not counted toward extra space complexity.

## Solution

```python
class Solution(object):
    def findDisappearedNumbers(self, nums):
        """
        :type nums: List[int]
        :rtype: List[int]
        """
        for num in nums:
            index = abs(num) - 1

            if nums[index] > 0:
                nums[index] *= -1

        missing_nums = []

        for i in range(len(nums)):
            if nums[i] > 0:
                missing_nums.append(i + 1)

        return missing_nums
```

## Example Walkthrough

### Input

```text
nums = [4,3,2,7,8,2,3,1]
```

### Mark Visited Indices

```text
4 -> mark index 3
3 -> mark index 2
2 -> mark index 1
7 -> mark index 6
8 -> mark index 7
2 -> index 1 already marked
3 -> index 2 already marked
1 -> mark index 0
```

Array becomes:

```text
[-4,-3,-2,-7,8,2,-3,-1]
```

### Find Positive Values

```text
Index 4 -> value 8  -> number 5 is missing
Index 5 -> value 2  -> number 6 is missing
```

### Output

```text
[5, 6]
```

## Interview Explanation

> Since the numbers are restricted to the range `1` to `n`, each value can be mapped directly to an index in the array. By marking the corresponding index as negative, we record that the number has been seen. After processing the entire array, any index that still contains a positive value corresponds to a missing number. This approach achieves **O(n)** time complexity while using **O(1)** extra space by modifying the input array in place.
