# 1700. Number of Students Unable to Eat Lunch

## Intuition

Students are standing in a queue, and sandwiches are in a stack.

At every step:

- If the student at the front of the queue prefers the sandwich on top, they take it and leave.
- Otherwise, the student moves to the back of the queue.

A direct simulation using a queue works, but there is an important observation:

> The order of the students does not matter.

The only thing that matters is:

- How many students prefer circular sandwiches (`0`)
- How many students prefer square sandwiches (`1`)

As long as there is at least one student who prefers the current sandwich, that sandwich will eventually be eaten because students can keep rotating in the queue.

The process only stops when the top sandwich's type is not preferred by any remaining student.

---

# Key Observation

We do **not** need to simulate the queue.

Instead:

1. Count how many students prefer `0`.
2. Count how many students prefer `1`.
3. Process the sandwiches one by one.
4. If no student prefers the current sandwich, stop immediately.

---

# Approach

1. Count the number of students preferring `0` and `1`.
2. Iterate through the `sandwiches` array.
3. For each sandwich:
   - If there are no students left who want that sandwich, return the number of remaining students.
   - Otherwise, decrease the count of students who wanted that sandwich.
4. If all sandwiches are consumed, return `0`.

# Optimal Solution

```python
class Solution(object):
    def countStudents(self, students, sandwiches):
        """
        :type students: List[int]
        :type sandwiches: List[int]
        :rtype: int
        """
        count = [0, 0]

        for student in students:
            count[student] += 1

        for sandwich in sandwiches:
            if count[sandwich] == 0:
                return count[0] + count[1]

            count[sandwich] -= 1

        return 0
```

---

# Complexity Analysis

### Time Complexity: `O(n)`

- One pass to count student preferences.
- One pass through the sandwiches.

```text
O(n + n) = O(n)
```

---

### Space Complexity: `O(1)`

The `count` array always contains exactly two elements:

```text
count = [students_wanting_0, students_wanting_1]
```

Therefore:

```text
Space Complexity = O(1)
```

---

# Why Doesn't Queue Simulation Matter?

Suppose:

```text
students = [1,0,1,0]
sandwich = 0
```

Even if the first student doesn't want it:

```text
1 moves to back
0 takes sandwich
```

The queue keeps rotating until:

- someone takes the sandwich, or
- nobody wants it.

Therefore, the order of students is irrelevant.

Only the counts of `0` and `1` matter.

# Alternative Approach (Simulation)

You can also solve this using:

- Queue (`deque`)
- Stack simulation

But that approach takes:

```text
Time Complexity: O(n²)
Space Complexity: O(n)
```

The counting approach is the optimal solution and the one typically expected in interviews.
