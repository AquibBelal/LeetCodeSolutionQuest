# 1475. Final Prices With a Special Discount in a Shop

## Optimal Approach: Monotonic Stack

### Intuition

For every item `prices[i]`, we need to find the **first price to its right** that is:

```text
<= prices[i]
```

This is a classic **Next Smaller or Equal Element** problem, which can be solved efficiently using a **Monotonic Increasing Stack**.

The stack will store the **indices** of items whose discount has not been found yet.

Whenever we encounter a price that is less than or equal to the price at the top index of the stack, we have found the discount for that item.

---

## Approach

1. Create a copy of `prices` called `ans`.
2. Create an empty stack to store indices.
3. Traverse the array from left to right.
4. While:
   - the stack is not empty, and
   - the current price is less than or equal to the price at the top index of the stack,
   
   then:
   - pop the index from the stack,
   - apply the discount to that index.
5. Push the current index onto the stack.
6. Return `ans`.

---

## Why Does This Work?

The stack always contains indices whose discounts have not yet been determined.

When we encounter:

```python
prices[i] <= prices[stack[-1]]
```

we know that:

- `prices[i]` is the **first smaller or equal element to the right**
- therefore it is the discount for `prices[stack[-1]]`.

After applying the discount, that index is removed from the stack.

## Solution

```python
class Solution(object):
    def finalPrices(self, prices):
        """
        :type prices: List[int]
        :rtype: List[int]
        """
        ans = prices[:]
        stack = []

        for i in range(len(prices)):
            while stack and prices[i] <= prices[stack[-1]]:
                index = stack.pop()
                ans[index] -= prices[i]

            stack.append(i)

        return ans
```

---

## Complexity Analysis

### Time Complexity: `O(n)`

Although there is a nested `while` loop, each index is:

- pushed onto the stack exactly once,
- popped from the stack exactly once.

Therefore:

```text
Total operations = Pushes + Pops
                 = n + n
                 = 2n
                 = O(n)
```

---

### Space Complexity: `O(n)`

In the worst case:

```text
prices = [1,2,3,4,5]
```

No element ever gets popped, so all indices remain in the stack.

Therefore:

```text
Space Complexity = O(n)
```
