# 232. Implement Queue using Stacks

## Intuition

A queue follows the **FIFO (First In, First Out)** principle:

```text
Push:  1, 2, 3
Pop:   1, then 2, then 3
```

A stack follows the **LIFO (Last In, First Out)** principle:

```text
Push:  1, 2, 3
Pop:   3, then 2, then 1
```

Since their orders are opposite, we can use **two stacks** to simulate a queue.

---

# Approach 1: Two Stacks (Amortized O(1))

We maintain:

- `input_stack`: used for `push()`
- `output_stack`: used for `pop()` and `peek()`

## Key Idea

### Push

Always push into `input_stack`.

### Pop / Peek

If `output_stack` is empty:

- Move all elements from `input_stack` to `output_stack`.

This reverses the order:

```text
input_stack  = [1,2,3]
output_stack = [3,2,1]

Top of output_stack = 1
```

which is exactly the front of the queue.

---

# Example

## Push 1

```text
input  = [1]
output = []
```

---

## Push 2

```text
input  = [1,2]
output = []
```

---

## Push 3

```text
input  = [1,2,3]
output = []
```

---

## Pop

Move all elements:

```text
input  = []
output = [3,2,1]
```

Pop:

```text
1
```

Remaining:

```text
output = [3,2]
```

---

## Peek

Top of output:

```text
2
```

---

# Why Does This Work?

Elements are reversed twice:

### First order

```text
1, 2, 3
```

### After moving

```text
3, 2, 1
```

The oldest element (`1`) is now on top of `output_stack`, giving us FIFO behavior.

---

# Solution

```python
class MyQueue(object):

    def __init__(self):
        self.input_stack = []
        self.output_stack = []

    def push(self, x):
        """
        :type x: int
        :rtype: None
        """
        self.input_stack.append(x)

    def pop(self):
        """
        :rtype: int
        """
        self._move()

        return self.output_stack.pop()

    def peek(self):
        """
        :rtype: int
        """
        self._move()

        return self.output_stack[-1]

    def empty(self):
        """
        :rtype: bool
        """
        return len(self.input_stack) == 0 and len(self.output_stack) == 0

    def _move(self):
        if not self.output_stack:
            while self.input_stack:
                self.output_stack.append(
                    self.input_stack.pop()
                )
```

---

# Complexity Analysis

## Push

```text
O(1)
```

because we simply append to `input_stack`.

---

## Pop

Worst case:

```text
O(n)
```

when all elements need to be transferred.

However, each element is:

- pushed once,
- moved once,
- popped once.

Therefore:

### Amortized Complexity

```text
O(1)
```

---

## Peek

Amortized:

```text
O(1)
```

---

## Empty

```text
O(1)
```

---

## Space Complexity

```text
O(n)
```

because both stacks together store at most `n` elements.

---

# Amortized Analysis

Suppose:

```text
push(1)
push(2)
push(3)
pop()
pop()
pop()
```

Transfers happen only once:

```text
input  -> output
```

Each element moves:

```text
input
↓
output
↓
removed
```

at most one time.

Therefore, over `n` operations:

```text
Total work = O(n)
Average per operation = O(1)
```

# Approach 2: One Stack Push-Costly

Another approach:

### Push

1. Move all elements to a temporary stack.
2. Push the new element.
3. Move everything back.

Then:

```text
pop()  -> O(1)
peek() -> O(1)
push() -> O(n)
```

This is usually not the preferred solution.
