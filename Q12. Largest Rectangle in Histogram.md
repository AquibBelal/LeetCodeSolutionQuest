# 84. Largest Rectangle in Histogram

## Intuition

For each bar in the histogram, suppose we choose that bar's height as the height of a rectangle.

To maximize the area for that bar, we need to know:

1. The **first smaller bar on the left**.
2. The **first smaller bar on the right**.

Everything between these two smaller bars can be included in the rectangle.

If:

```text
height = heights[i]
left = index of previous smaller element
right = index of next smaller element
```

then:

```text
width = right - left - 1
area = height * width
```

Computing the previous and next smaller elements for every bar can be done efficiently using a **Monotonic Increasing Stack**.

---

# Approach 1: Previous Smaller + Next Smaller Arrays

## Algorithm

1. Compute the index of the previous smaller element for every bar.
2. Compute the index of the next smaller element for every bar.
3. For every bar:
   - Calculate the width.
   - Calculate the area.
   - Update the maximum area.

---

## Solution (Two Arrays)

```python
class Solution(object):
    def largestRectangleArea(self, heights):
        n = len(heights)

        prev_smaller = [-1] * n
        next_smaller = [n] * n
        stack = []

        # Previous smaller
        for i in range(n):
            while stack and heights[stack[-1]] >= heights[i]:
                stack.pop()

            if stack:
                prev_smaller[i] = stack[-1]

            stack.append(i)

        stack = []

        # Next smaller
        for i in range(n - 1, -1, -1):
            while stack and heights[stack[-1]] >= heights[i]:
                stack.pop()

            if stack:
                next_smaller[i] = stack[-1]

            stack.append(i)

        max_area = 0

        for i in range(n):
            width = next_smaller[i] - prev_smaller[i] - 1
            area = heights[i] * width
            max_area = max(max_area, area)

        return max_area
```

---

## Complexity Analysis

- **Time Complexity:** `O(n)`
- **Space Complexity:** `O(n)`

# Optimal Approach (Single Stack)

Instead of explicitly computing the previous and next smaller arrays, we can compute the area immediately when we find the next smaller element.

---

## Key Idea

Whenever we encounter a bar that is **smaller** than the bar at the top of the stack:

```python
heights[i] < heights[stack[-1]]
```

we know:

- The current index `i` is the **next smaller element**.
- The new stack top after popping is the **previous smaller element**.

Therefore, we can compute the rectangle area immediately.

---

## Algorithm

1. Maintain a monotonic increasing stack of indices.
2. Traverse the histogram.
3. Whenever the current bar is smaller than the top of the stack:
   - Pop the top.
   - Compute its width.
   - Compute its area.
4. Push the current index.
5. To process all remaining bars, append a virtual bar of height `0`.

---

## Optimal Solution

```python
class Solution(object):
    def largestRectangleArea(self, heights):
        stack = []
        max_area = 0

        heights.append(0)

        for i in range(len(heights)):
            while stack and heights[i] < heights[stack[-1]]:
                height = heights[stack.pop()]

                if stack:
                    width = i - stack[-1] - 1
                else:
                    width = i

                max_area = max(max_area, height * width)

            stack.append(i)

        return max_area
```

---

## Why Does the Width Formula Work?

Suppose:

```text
stack = [1,2,3]
```

and we pop:

```text
index = 3
height = heights[3]
```

After popping:

```text
stack[-1]
```

becomes the previous smaller element.

The current index `i` is the next smaller element.

Therefore:

```text
width =
next_smaller
-
previous_smaller
-
1
```

which becomes:

```python
width = i - stack[-1] - 1
```

If the stack becomes empty:

```text
previous_smaller = -1
```

Therefore:

```python
width = i
```

## Complexity Analysis

### Time Complexity: `O(n)`

Each index is:

- pushed once,
- popped once.

Therefore:

```text
2n operations
```

which simplifies to:

```text
O(n)
```

---

### Space Complexity: `O(n)`

In the worst case:

```text
heights = [1,2,3,4,5]
```

all indices remain in the stack.

Therefore:

```text
Space Complexity = O(n)
```
